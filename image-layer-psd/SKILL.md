# image-layer-psd

## Purpose

Use this skill when the user wants a generated image or source image turned into Photoshop-editable layers: subject, background, text, logo, decoration, shadows, or other named production layers.

This skill does not claim that the official OpenAI image API natively returns PSD. Official image generation outputs image formats such as PNG/JPEG/WebP. This workflow uses GPT Image / image-2 repeatedly to produce separate PNG layers, then assembles those PNGs into a real multi-layer PSD locally.

The bundled CLI is [scripts/image_layer_psd_cli.js](scripts/image_layer_psd_cli.js). Run it from the skill root:

```bash
node scripts/image_layer_psd_cli.js run ...
```

## A-E Workflow

1. **[A] Define layer contract**  
   Pick a preset or explicit layer list before generating anything. Read [references/layer-contracts.md](references/layer-contracts.md) when the use case is unclear.

2. **[B] Apply fixed prompt templates**  
   Convert the user's request into prompts for each layer: transparent subject, completed background, isolated text, isolated logo, and optional decoration/shadow layers. Read [references/prompt-templates.md](references/prompt-templates.md) before changing prompts.

3. **[C] Generate PNG layers with image-2**  
   Generate a base PNG if the user provides only a requirement. If the user provides an image, use it as the base. Then generate one transparent PNG per layer. Do not ask image-2 to return PSD.

4. **[D] Build and QC PSD**  
   Assemble PNG layers into PSD locally, then run QC: PSD signature, layer count, layer names, image dimensions, alpha channel, subject transparency, and background holes.

5. **[E] Retry failed layers**  
   If QC fails, retry only the failed layer when possible using `--previous-failure`. Retry the whole split when background completion or subject contamination affects multiple layers.

## CLI Quick Start

Full run from a prompt:

```bash
node scripts/image_layer_psd_cli.js run \
  --mode official \
  --permission-code "$OPENAI_API_KEY" \
  --prompt "电商床品主图，真实摄影质感，模特躺在床上，暖色自然光" \
  --preset ecommerce \
  --out-dir output/bedding-layers \
  --output output/bedding.psd
```

Full run from an existing image:

```bash
node scripts/image_layer_psd_cli.js run \
  --mode official \
  --permission-code "$OPENAI_API_KEY" \
  --image /absolute/path/source.png \
  --prompt "Keep the product identity, separate editable production layers" \
  --preset product \
  --out-dir output/product-layers \
  --output output/product.psd
```

Build PSD from existing layer PNGs:

```bash
node scripts/image_layer_psd_cli.js build-psd \
  --layers-json output/product-layers/layers.json \
  --output output/product.psd
```

Run QC:

```bash
node scripts/image_layer_psd_cli.js qc \
  --layers-json output/product-layers/layers.json \
  --psd output/product.psd
```

## Commands

- `generate-base`: create `base.png` from prompt or edit/reference image.
- `split-layers`: generate one PNG per required layer and write `layers.json`.
- `build-psd`: assemble PNG layers into a real multi-layer PSD.
- `qc`: return a JSON quality report and retry reason.
- `run`: execute base generation, splitting, PSD assembly, QC, and limited retries.

## Output Rules

Always report:

- PSD path
- layer directory
- layer names
- QC result
- retry reason if QC failed
- whether the PSD was locally assembled

Never print API keys, permission codes, provider keys, or proxy tokens.

## Quality Boundaries

- A locally assembled PSD is real multi-layer PSD only if each layer PNG is separate and named.
- Text/logo layers generated by image-2 may be raster layers, not editable vector/text layers.
- If exact typography, brand marks, or product labels matter, tell the user to inspect and adjust in Photoshop.
- If official generation cannot preserve an original logo/text accurately, recommend passing a source image and retrying the affected layer.
